javascript学习笔记-01

发表于 2022-06-22 更新于 2022-06-24 分类于 web前端

arguments的使用

  • 不确定实参个数时，使用arguments获取所有参数
function fn(arguments){
    console.log(arguments);
}
fn(1,2,3);

arguments并不是真正意义上的数组：

  1. 具有length
  2. 以索引进行存储
  3. 没有pop()与push()等数组方法

javascript作用域

  • javascript作用域分为全局作用域与局部作用域

全局作用域

  • 整个script标签可用,或单独的js文件可用
<script>
    var num = 10;
    function fn(){
        console.log(num); //可在函数中使用
    }
</script>

局部作用域

  • 仅在函数内部起作用
<script>
    function fn(){
        var num = 10;
        console.log(num);
    }
    console.log(num); //num is not defined
</script>

在函数内部未经声明，直接赋值的变量也是全局变量

<script>
    function fn(){
        num = 10;
    }
    console.log(num); // 10
</script>

变量的销毁

  • 全局变量：浏览器关闭时才会销毁。
  • 局部变量：代码块结束后，就会被销毁。

作用域链

当函数中嵌套了函数，作用域链就产生了。

var num = 10;
function fn1(){
    var num = 20;
    function fn2(){
        console.log(num); //20
    }
}

js运行步骤

  • javascript解析器在运行javascript语句时分两步：预解析和代码执行。

预解析

  • 预解析分为变量预解析（变量提升）和函数预解析（函数提升）。

  • js解析器会将所有的var与function提升到当前作用域的最前面。

变量预解析

不提升赋值操作

console.log(num);
var num = 10;
//相当于以下代码
var num;
console.log(num); //undefined
num = 10;

函数预解析

fn();
var fn = function(){
    console.log(123);
}
//相当于以下代码
var fn;
fn();
fn = function(){
    console.log(123);
}

代码执行

  • 代码按照书写顺序，由上到下依次执行
